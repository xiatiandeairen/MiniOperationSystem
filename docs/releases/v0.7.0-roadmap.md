# MiniOS v0.7.0 Roadmap — "See Every Step"

## 版本演进

```
v0.1–v0.6  基础设施 → 教学命令 → 可视化 → 对比实验 → 故障 → 学习路径
v0.7.0     日志体系 — 让每一层的行为对学习者透明可查
```

## 为什么做

v0.6.0 的 `trace follow` 可以看到一次命令的调用链，但粒度太粗——只知道"调用了 vfs_open"，不知道"vfs_open 内部查找了哪些 inode、锁了哪个 Mutex、花了多少时间"。学习者需要一个**可调粒度**的观测系统：平时看 INFO 级概要，深入时开 DEBUG 看明细。

## 特性规划

### F1: 内核日志系统 — `klog!` 宏

分级日志宏，运行时可过滤：

```rust
klog!(ERROR, "memory", "frame allocation failed: no free frames");
klog!(WARN,  "scheduler", "boost triggered at tick {}", tick);
klog!(INFO,  "fs", "mounted ramfs at /");
klog!(DEBUG, "memory", "allocate_frame: scanning from byte {}", idx);
klog!(TRACE, "ipc", "message queue {} send: {} bytes", qid, len);
```

Shell 中控制：
```
MiniOS $ log level debug        → 显示 DEBUG 及以上
MiniOS $ log level info         → 只显示 INFO 及以上（默认）
MiniOS $ log module memory      → 只显示 memory 模块日志
MiniOS $ log module all         → 显示所有模块
MiniOS $ log off                → 关闭日志输出
```

### F2: trace 关键字过滤 — `trace filter`

```
MiniOS $ trace filter syscall   → 只看 module=syscall 的 span
MiniOS $ trace filter fs        → 只看 module=fs 的 span
MiniOS $ trace filter all       → 不过滤（默认）
MiniOS $ trace follow ls / --filter fs  → 只看 fs 层的 trace
```

### F3: 子系统日志埋点

在关键路径添加 `klog!` 调用：
- memory: frame alloc/dealloc, page map/unmap, heap alloc
- fs: open/read/write/close 细节, inode lookup
- scheduler: tick decision, task switch, boost
- syscall: entry/exit + 参数
- ipc: send/receive

### F4: `log history` — 查看最近日志

```
MiniOS $ log history 20       → 最近 20 条日志
MiniOS $ log history --level error → 只看错误
```

Ring Buffer 存储最近 256 条日志。

### F5: `debug` 命令 — 一键开启详细模式

```
MiniOS $ debug on     → log level trace + trace follow 自动开启
MiniOS $ debug off    → 恢复 log level info
```

### F6: explain log — 解释日志系统

```
MiniOS $ explain log
=== How the logging system works ===
5 levels: ERROR > WARN > INFO > DEBUG > TRACE
...
```

## 优先级

| 优先级 | 特性 | 理由 |
|--------|------|------|
| P0 | F1 klog! 宏 + log 命令 | 日志系统的基础设施 |
| P0 | F3 子系统埋点 | 没有埋点日志系统就是空壳 |
| P1 | F2 trace filter | 增强已有 trace 的可用性 |
| P1 | F4 log history | 让日志可回看 |
| P2 | F5 debug 命令 | 便捷开关 |
| P2 | F6 explain log | 教学配套 |
