# MiniOS v0.2.0 — "Teach by Running"

**发布日期**: 2026-02-26
**代号**: "Teach by Running"

---

## 版本概览

v0.2.0 将 MiniOS 从"能跑的 OS"升级为"能教人 OS 的工具"。新增 8 个教学导向的 Shell 命令，让用户可以亲手操作进程、观察调度器、探索内存布局、追踪系统调用链路。

## 新增特性

### 调度可观测 (P0)
| 命令 | 功能 |
|------|------|
| `spawn <name>` | 创建一个内核后台任务 |
| `kill <pid>` | 终止指定进程 |
| `sched` | 显示 MLFQ 各队列状态和当前运行任务 |
| `nice <pid> <prio>` | 修改进程优先级 (0-3) |

### Trace 教学化 (P1)
| 命令 | 功能 |
|------|------|
| `trace follow <cmd>` | 执行命令并只显示该命令产生的 trace 链路 |
| `trace tree` | 树形显示 trace span 层级 + 教学注解 |

**Syscall & VFS trace 集成**: 每个 syscall (read/write/open/close/exit/getpid/yield/uptime/meminfo) 和 VFS 操作 (vfs_open/vfs_read/vfs_write/vfs_close/vfs_mkdir/vfs_stat...) 自动产生 trace span。

**教学注解**: `trace tree` 和 `trace follow` 输出附带人类可读的解释，例如:
- `[syscall] sys_open ← opens a file by path`
- `[fs] vfs_read ← VFS delegates to driver`

### 内存可见 (P2)
| 命令 | 功能 |
|------|------|
| `pagetable <addr>` | 分解虚拟地址的 4 级页表索引 (PML4/PDPT/PD/PT/offset) |
| `frames` | 可视化帧分配状态 (进度条 + 百分比) |
| `alloc <size>` | 手动分配堆内存并显示地址和堆统计 |

## Bug 修复

## 构建与运行

```bash
cargo make run-gui    # 推荐: 图形窗口
cargo make run        # 无头模式
```
