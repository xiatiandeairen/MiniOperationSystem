# MiniOS v0.6.0 Roadmap — "Connect the Dots"

## 版本演进

```
v0.1.0  能跑        → 基础设施           (15 cmds, 6900 lines)
v0.2.0  能教        → 操控命令           (23 cmds, 7300 lines)
v0.3.0  能看见      → explain + /proc    (27 cmds, 7500 lines)
v0.4.0  能感受      → compare + lab      (33 cmds, 8200 lines)
v0.5.0  能破坏      → crash + scripts    (38 cmds, 8700 lines)
v0.6.0  能串联      → 管道 + 学习路径     (目标: 42 cmds, ~9500 lines)
```

## 为什么做这一版

v0.1–v0.5 积累了 38 个命令、5 个 lab、5 个 crash、5 个 compare、9 个 explain。但用户面前摆着一座命令迷宫——**不知道从哪开始、按什么顺序、做完了没有**。

v0.6.0 解决的核心问题：**把离散的点连成线**。

具体手段：
1. **管道** `|` 让命令可以组合，从"逐个使用"到"组合使用"
2. **journey** 学习路径把 38 个命令编排成 4 章节的渐进式课程
3. **graduation** 完成全部学习后输出总结报告——给用户成就感闭环
4. **trace 端到端** 打通 trace export → 文件 → trace-viewer 加载的数据通路

## 特性需求（精确定义）

---

### F1: 管道操作符 `|`

**用户故事**: 作为学习者，我想用 `cat /proc/meminfo | grep Free` 过滤输出，这样我可以体验 Unix 的"小工具组合"哲学。

**验收标准**:
- [ ] `echo hello world | wc` 输出 `1 3 12`
- [ ] `cat /proc/meminfo | grep Free` 只输出包含 "Free" 的行
- [ ] `ps | head 2` 只输出前 2 行
- [ ] 无管道时行为不变
- [ ] 只支持单级管道（`a | b`），不支持多级（`a | b | c`）

**实现方案**:
- Shell 解析时检测 `|` 分隔符
- 左侧命令的 `println!` 输出重定向到内存 Vec<u8>（替换全局 print 目标）
- 右侧命令从该 Vec 读取输入（作为"虚拟文件"）
- head/grep/wc 需要支持 stdin 模式（无文件参数时从管道读取）

**风险**: println! 是全局宏，重定向需要临时替换 framebuffer/VGA 的输出目标。可能需要一个 thread-local 输出缓冲。

---

### F2: 学习路径引擎 — `journey` 命令

**用户故事**: 作为新用户，我启动 MiniOS 后想知道"我应该按什么顺序学习"，并在完成每一步后看到进度更新。

**验收标准**:
- [ ] `journey` 显示 4 个章节，每章节 4-5 个步骤
- [ ] 已完成的步骤显示 ✅，未完成显示 ⬜
- [ ] 每个章节显示完成百分比进度条
- [ ] 完成步骤后再次运行 `journey` 能看到更新
- [ ] 显示"下一步推荐"

**章节设计**:
```
Chapter 1: First Steps (入门)
  - help, ps, meminfo, ls /, uptime

Chapter 2: Look Inside (原理)
  - explain ls, cat /proc/meminfo, trace follow ls /, pagetable

Chapter 3: Hands-on Experiments (实验)
  - lab scheduler-fairness, lab memory-usage, crash oom, compare scheduler

Chapter 4: Power User (进阶)
  - spawn worker, run /etc/init.sh, grep Free /proc/meminfo, trace stats
```

**实现方案**:
- 全局 bitmap `u64` 追踪已完成步骤（最多 64 步，够用）
- 每个命令 handler 末尾调用 `journey::mark(STEP_ID)`
- `journey` 命令读取 bitmap 渲染进度

---

### F3: 毕业报告 — `graduation` 命令

**用户故事**: 完成全部 4 章后，我想看到一个总结，确认"我学了哪些 OS 概念"。

**验收标准**:
- [ ] 完成 <50% 步骤时显示"继续加油"
- [ ] 完成 100% 步骤时显示完整毕业报告
- [ ] 报告列出每章学到的 OS 概念

**输出示例**:
```
MiniOS $ graduation
🎓 Congratulations! You completed the MiniOS learning journey!

Concepts you explored:
  ✅ Process management — creating, scheduling, signaling processes
  ✅ Memory management — physical frames, virtual pages, heap allocation
  ✅ Filesystem — VFS abstraction, inodes, file descriptors
  ✅ Tracing — span-based observability, ring buffer storage
  ✅ Fault handling — OOM, stack overflow, page faults
  ✅ OS design trade-offs — MiniOS vs Linux comparisons

You ran 16/16 learning steps across 4 chapters.
Total commands executed: 47 (from history)
```

### F4: Trace 端到端数据通路

**用户故事**: 我想在 Shell 中运行 `trace export`，然后在浏览器的 trace-viewer 中看到真实的内核 trace 数据。

**验收标准**:
- [ ] `trace export` 输出有效 JSON 到串口
- [ ] `cargo make run-trace` 捕获串口输出到文件
- [ ] 捕获的 JSON 可被 `trace-viewer/index.html` 加载和渲染
- [ ] 更新 `scripts/capture-trace.sh` 提取 JSON 部分

**实现方案**:
- trace export 输出前后加特殊标记（`---TRACE-BEGIN---` / `---TRACE-END---`）
- capture-trace.sh 用 sed 提取标记之间的内容
- trace-viewer 的 parser 兼容真实数据格式

---

### F5: alias 命令

**验收标准**:
- [ ] `alias ll ls /` 注册别名
- [ ] 输入 `ll` 执行 `ls /`
- [ ] `alias` (无参数) 列出所有别名
- [ ] 最多 16 个别名

---

### F6: 测试补充

**验收标准**:
- [ ] Shell parser 管道分割测试
- [ ] journey bitmap 测试
- [ ] alias 存储/查找测试
- [ ] 测试总数 ≥70

## 不做的事
- 多级管道 `a | b | c` — 单级够教学
- Tab 补全 — 实现复杂（需 raw mode 键盘处理），延后
- SDK 拆分 — 先把产品体验做好，SDK 是 v0.7+ 的事
- /proc/self — 优先级不够

## 优先级调整（相对 v0.6.0 初版 roadmap 的变化）

| 特性 | 初版优先级 | 调整后 | 原因 |
|------|-----------|--------|------|
| 管道 | P0 | **P0** | 不变 — Unix 哲学核心 |
| journey | P0 | **P0** | 不变 — 解决用户迷失问题 |
| SDK 提取 | P1 | **降为 v0.7+** | 先做产品体验，SDK 是锦上添花 |
| Tab 补全 | P2 | **降为 v0.7+** | 实现复杂，ROI 不够 |
| alias | P2 | **P1** | 简单且实用，配合管道 |
| graduation | 新增 | **P0** | 学习闭环的关键一环 |
| trace 端到端 | 新增 | **P1** | v0.3 就该做，拖了 3 版 |
| 测试补充 | 常规 | **P2** | 保持质量底线 |

## 版本目标

v0.6.0 = **管道 + 学习路径 + 毕业报告 + trace 端到端 + alias**

让用户从"每次随便敲几个命令"到"按路径完成一门微型 OS 课程，最后拿到毕业总结"。
