# MiniOS v0.1.0 — 首个可交互版本

**发布日期**: 2026-02-26
**代号**: "Hello, MiniOS!"

---

## 版本概览

从空仓库到可引导、可交互的微型操作系统内核。用户可以通过 `cargo make run-gui` 一键启动 QEMU，在图形窗口中看到彩色引导横幅和 Shell 提示符，输入命令与 OS 交互。

## 数字摘要

| 指标 | 数值 |
|------|------|
| Rust 源文件 | 66 |
| 代码行数 | ~6,800 |
| Cargo Crate 数 | 13 + 1 构建工具 |
| 单元测试 | 59 |
| Shell 命令 | 15 |
| 系统调用 | 7 |
| Trace 容量 | 4,096 span |
| 支持内存 | 256 MiB (65,536 frames) |

## 功能特性

### 引导与显示
- BIOS 引导 (bootloader_api v0.11)
- 帧缓冲像素文本控制台 (8×16 bitmap font, 160×45 字符)
- 彩色输出 (绿色提示符, 红色错误, 默认淡蓝)
- ASCII art 引导横幅

### 内存管理
- Bitmap 物理帧分配器 (O(n) 首次适配)
- 4 级页表虚拟内存映射 (map/unmap/translate)
- 1 MiB 链表堆分配器 (`#[global_allocator]`)

### 进程与调度
- 进程控制块 (PCB) + PID 分配器
- 上下文切换汇编 (global_asm!, callee-saved 寄存器)
- MLFQ 4 级调度器 (时间片 2/4/8/16, 优先级 boost)

### 文件系统
- VFS 虚拟文件系统层
- RamFS 内存文件系统 (inode-based, BTreeMap)
- ProcFS 实时虚拟文件 (/proc/meminfo, /proc/uptime)
- 文件描述符表 (最大 256 FD)

### 系统调用
- 分发器 (read/write/getpid/yield/exit/uptime/meminfo)
- 当前为内核态函数调用 (非 int 0x80 陷入)

### IPC 进程间通信
- 消息队列 (16 队列, 256 字节载荷)
- TraceContext 跨进程传播字段

### Shell 交互终端
- 15 个内置命令: help, echo, clear, uptime, meminfo, ls, cat, mkdir, touch, write, pwd, ps, trace (list/tree/stats/clear/export)
- PS/2 键盘输入 + 串口备选输入
- 实时字符回显

### Trace 追踪系统
- 4,096 slot Ring Buffer 存储
- SpanGuard RAII 自动关闭 span
- trace_span! / trace_event! 宏
- JSON 序列化导出 (trace export 命令)
- 引导流程 trace 集成

### Trace Viewer 可视化
- 浏览器端 HTML/CSS/JS 瀑布图
- 23 span 样例数据
- 文件拖拽加载

### 中断处理
- IDT (6 个异常处理: breakpoint, double fault, page fault, GPF, ...)
- PIC 8259 (timer IRQ0, keyboard IRQ1)
- 定时器回调机制

## 架构设计

```
Shell → Syscall → Process/FS/IPC → Memory → HAL → Hardware
                      ↑
               Trace (横切所有层)
```

- **Trait 解耦**: 8 个 trait 合约定义子系统边界
- **KernelServices**: 全局注册表管理子系统实例
- **Workspace 隔离**: 13 个 crate 各自独立编译和测试

## 已知限制

| 项目 | 说明 | 优先级 |
|------|------|--------|
| Syscall 非陷入式 | 函数调用而非 int 0x80 | 中 |
| 无用户态隔离 | 所有代码运行在 Ring 0 | 低 |
| 上下文切换未接入调度 | Shell 运行在 kernel_main, 任务不被抢占 | 中 |
| FD 表全局共享 | 应每进程独立 | 低 |
| FileStat 时间戳为 0 | 未接入 TSC | 低 |
| alloc 热路径无 OOM 处理 | 堆耗尽时 panic | 低 |

## 构建与运行

```bash
cargo make run-gui    # 图形窗口 (推荐)
cargo make run        # 无头模式 (串口)
cargo make test       # 59 个单元测试
cargo make ci         # fmt + clippy + test
```
