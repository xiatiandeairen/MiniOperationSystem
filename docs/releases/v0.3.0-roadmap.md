# MiniOS v0.3.0 Roadmap — "See the Machine"

## 三重视角分析

### 用户痛点（学习者视角）
1. **看不到系统内部发生了什么** — 输入 `ls` 后只看到结果，不知道经过了几层调用、哪些锁被获取
2. **概念停留在文字** — 页表、中断、上下文切换都是抽象概念，没有直观的"摸到"感
3. **没有实验反馈循环** — 无法做实验式学习（改参数→看效果→理解原理）
4. **启动太慢** — 15秒引导等待，打断学习节奏

### 专家视角（OS 教学者）
1. 好的OS教学工具需要**explain mode** — 不只执行，还要解释每一步在做什么
2. 调度器参数应该**可调可看** — 修改时间片大小后立刻看到调度行为变化
3. 中断处理是教学难点 — 需要可视化中断到达→处理→返回的完整流程
4. 页表遍历是记忆难点 — 需要实际执行地址翻译并展示每级的内容

### 成熟软件视角（Linux/xv6/教学OS参考）
1. xv6 的价值在于**代码可读** — 每个模块 < 500 行，有大量注释
2. Linux 的 `/proc` 和 `/sys` 是最好的内核教学界面 — 一切状态皆文件
3. 教学OS需要**引导式体验** — 第一次启动有 tutorial 引导用户做什么
4. dtrace/ftrace 的价值在于**零成本观测** — 不需要修改代码就能看到行为

## v0.3.0 目标

> 让用户第一次启动 MiniOS 时，10分钟内理解"一次 ls 命令是如何从键盘输入到屏幕输出"的完整链路。

## 特性规划

### F1: 引导式教程 (tutorial)
Shell 启动后显示互动教程，引导用户逐步探索 OS：

```
Welcome to MiniOS! Let's explore how an OS works.

Step 1: Type 'help' to see available commands.
Step 2: Type 'ps' to see running processes.
Step 3: Type 'trace follow ls /' to see what happens inside a command.
Step 4: Type 'explain ls' to learn how the filesystem works.

Type 'tutorial' at any time to restart this guide.
```

### F2: explain 模式 — 命令级教学
`explain <command>` 不执行命令，而是解释它会做什么：

```
MiniOS $ explain ls /
The 'ls' command lists directory contents. Here's what happens:

1. Shell parses "ls /" → command="ls", args=["/"]
2. SYSCALL: sys_open("/", READ) → VFS resolves path to root inode
3. SYSCALL: sys_read(fd) → VFS asks RamFS to list children
4. For each child: print name and size
5. SYSCALL: sys_close(fd) → releases file descriptor

Key concepts:
- VFS (Virtual File System) abstracts the actual storage
- Inodes identify files/directories by number, not name
- File descriptors are process-local handles to open files
```

### F3: 实时中断计数器
`interrupts` 命令显示各中断的触发次数和频率：

```
MiniOS $ interrupts
IRQ   NAME       COUNT    RATE
0     Timer      15234    100/s
1     Keyboard   47       on-demand
14    Page Fault  0        -
```

### F4: 调度器实验
`sched config` 显示和修改调度参数：

```
MiniOS $ sched config
MLFQ Configuration:
  Queue 0 [HIGH]: time_slice = 2 ticks
  Queue 1 [MED]:  time_slice = 4 ticks
  Queue 2 [LOW]:  time_slice = 8 ticks
  Queue 3 [IDLE]: time_slice = 16 ticks
  Boost interval: 100 ticks

MiniOS $ sched config boost 50
Boost interval changed: 100 → 50 ticks
```

### F5: 完善 /proc 虚拟文件系统
扩展 procfs，让所有内核状态可通过 `cat` 访问：

```
MiniOS $ cat /proc/interrupts    → 中断统计
MiniOS $ cat /proc/scheduler     → 调度器状态
MiniOS $ cat /proc/trace         → trace 统计
MiniOS $ cat /proc/version       → OS 版本信息
```

### F6: 优化启动速度
减少串口日志输出量，跳过不必要的 boot-time 测试。

## 不做的事
- 不做抢占式调度（复杂度高，教学价值可通过 cooperative 实现）
- 不做 int 0x80 真实陷入（函数调用式 syscall 对教学足够）
- 不做 SDK 拆分（当前优先用户体验）

## 优先级
1. F2 explain 模式（最高教学 ROI）
2. F5 /proc 扩展（低成本，高可见）
3. F3 中断计数器（直观理解硬件中断）
4. F1 引导式教程（首次体验）
5. F4 调度实验（进阶功能）
6. F6 启动优化（体验打磨）
