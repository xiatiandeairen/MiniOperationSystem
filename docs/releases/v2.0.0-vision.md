# MiniOS v2.0 Vision — "A Living OS"

## v1.0 回顾

v1.0 是一个**静态的教学橱窗**——用户可以看、可以查、可以做实验，但 OS 本身不会"活"起来。spawn 的进程不真正运行，写的文件重启就丢失，没有真正的并发。

## v2.0 定位

> 从"教学橱窗"到"活的操作系统"——进程真正并发执行，数据真正持久存在，系统真正在运行中被观测。

## 三大方向

---

### 方向一：真实多任务 — "进程活了"

**核心目标**: spawn 的进程真正在后台运行，Shell 不再独占 CPU。

```
MiniOS $ spawn counter
Created task 'counter' PID 2
MiniOS $ ps
PID  NAME     STATE    PRIO  CPU
0    idle     READY    3     5000
1    shell    RUNNING  0     4500   ← Shell 是一个 task
2    counter  READY    1     500    ← 真正在运行!
```

**技术路径**:
1. 将 Shell 注册为 PID 1 task（不再直接跑在 kernel_main）
2. 在 timer tick 中执行真实 switch_context_asm
3. 每个 task 有独立的 kernel stack
4. spawn 创建的 task entry function 被调度器选中后真正执行

**教学价值**: 用户可以亲眼看到两个进程交替获得 CPU 时间——这是 OS 最核心的概念。

**风险**: 高。需要解决 Shell 的 keyboard input 在上下文切换中不丢失的问题。

---

### 方向二：可观测的并发 — "看见调度发生"

**核心目标**: 用户不仅能看到进程列表，还能**实时看到调度决策**。

```
MiniOS $ watch ps 1
[tick 1000] PID 1 (shell) → PID 2 (counter) [preempt: timeslice]
[tick 1002] PID 2 (counter) → PID 1 (shell) [yield: IO wait]
[tick 1004] PID 1 (shell) → PID 0 (idle) [block: keyboard]
```

**特性**:
- `watch <cmd> <interval>` — 定时重复执行命令
- `trace live` — 实时流式显示 trace span
- `sched watch` — 实时显示每次调度决策
- 帧缓冲分屏：上半屏 Shell，下半屏实时监控

**教学价值**: 从"事后查看"到"实时观察"，让调度不再是抽象概念。

---

### 方向三：Web 集成 — "浏览器里的 OS"

**核心目标**: 将 MiniOS 的教学体验从 QEMU 扩展到浏览器。

**路径 A: Trace Viewer 增强**
- 实时串口 WebSocket 连接
- 自动渲染 kernel trace
- 交互式 span 详情面板

**路径 B: WebAssembly 模拟**
- 将核心子系统（scheduler, memory, fs）编译为 WASM
- 浏览器中运行教学模拟，无需 QEMU
- 降低入门门槛到零安装

**路径 C: 在线课程平台**
- 每个 lab/crash/compare 嵌入网页
- 自动评分 + 进度追踪
- 分享学习成果

---

## 优先级评估

| 方向 | 技术难度 | 教学价值 | 市场价值 | 建议 |
|------|---------|---------|---------|------|
| 真实多任务 | 🔴 高 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | 做——这是"真正的OS"的分水岭 |
| 可观测并发 | 🟡 中 | ⭐⭐⭐⭐ | ⭐⭐ | 做——依赖方向一 |
| Web 集成 | 🟡 中 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 路径B(WASM)最有市场价值 |

## v2.0 版本计划

```
v2.0-alpha: Shell 作为 task + 真实上下文切换
v2.0-beta:  spawn 真正执行 + watch 实时监控
v2.0-rc:    稳定性 + 文档 + 教学内容更新
v2.0:       发布
```

**不做的事** (v2.0 scope 之外):
- 用户态隔离 (Ring 3) — 太复杂，v3.0 考虑
- 磁盘持久化 — 需要块设备驱动，v3.0 考虑
- 网络栈 — 超出教学 OS 范围
- SMP 多核 — 教学价值不如单核做深

## 成功标准

v2.0 发布时，用户应该能做到：

```
MiniOS $ spawn counter     ← 创建后台任务
MiniOS $ ps                ← 看到 counter 的 CPU time 在增长
MiniOS $ watch ps 2        ← 每 2 秒刷新进程列表
MiniOS $ trace live        ← 看到 scheduler 在实时切换进程
MiniOS $ kill 2            ← 终止后台任务
MiniOS $ graduation        ← 毕业报告包含"真实并发体验"
```

这将是 MiniOS 从"教学工具"到"真正的操作系统"的质变。
